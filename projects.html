Current: Chopper Dropper

I began using Unity in 2013 and started with basic model import (from my halo modding days) and first person navigation with collision. I really enjoyed the aspect of 3D games but I didn't have a solid idea of what I'd want to pursue. When Unity came out with a simplified 2D project template, I had an idea but 2D wasn't my specialty. Using the 2D template, I made a quick mock-up with crude placeholder art for basic mechanics. I decided to take the concepts of 2D and the art of 3D to start creating a 2.5D game. My goal in this project is to not only make a complete retro arcade side scrolling shooter game with high replayability, but also to gain experience in efficient game design focusing on code reusability. This project is closed source at this time. Every now and then I update a small web demo here.

MIPS Simulator

For my graduate Computer Architecture class, I completed a solo project to program a MIPS simulator in the language of our choice (I chose C). The input takes a text file filled with 32 bit binary string that are MIPS machine instructions and accurately processes the instructions cycle by cycle through an 8-stage pipeline with port forwarding. After each cycle was executed, a snapshot was printed with various data such as program counter, pipeline status, stalled instructions, register values, memory, and branches taken. This project was the foundation to three years of Computer Architecture research using other simulators such as MacSim and GPGPU. Code is available on my GitHub.

Wii60 Project

From September 2011 to April 2012, I embarked on a two semester project. The original intention was to play the Xbox 360 with a Wiimote, but it quickly turned into a much larger project. I built a circuit that centered around an mBed that directly interfaced with the 2 joysticks, 2 triggers, and 16 buttons of a CG Wireless Xbox 360 controller. My final result included the Wiimote working and a keyboard and mouse along with the inputs for a Playstation 2 controller and Nintendo 64 controller. The purpose of the project soon came to be giving the ability for any input to an Xbox 360 controller. I hope to expand the project in the near future. Album can be viewed here. Code snippets can be found on my GitHub.

Hog Runner

Hog Runner is a game I programmed for my CMPS 455 class at Southeastern Louisiana University as an undergraduate. It uses the Allegro library with Mappy. The graphics come from a variety of free-to-use sprites spanning multiple sites. The sounds and background textures were directly ripped from Halo: Combat Evolved. While it could use a few tweaks, I prefer to leave it as is and treat it as a learning example. Download here. Code is available on my GitHub.

Halo XBE Modder

This is an application I wrote in C++/CLI for modifying .xbe files for Halo mods. This application let's you change the title, maps directory, and version number. The benefit of these changes let's you compile multiple Halo 1 mappacks into a single directory and doesn't suffer cache errors when switching between different mappacks. Source code available here.

RFID Access for Handicapped

For my undergraduate Computer Architecture class, I completed a group project using a mBed microcontroller to create a prototype of a lift that activates via RFID. A sabertooth dual motor driver was used to build the mock-up "Lego-sized" prototype. During the demo, various RFID cards were scanned and only the ones verified by a local static database caused the prototype to lift and lower.
